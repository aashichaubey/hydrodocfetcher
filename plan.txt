Tech stack

* Python
* FastAPI for the webhook/app
* Resend for receiving and sending emails
* Playwright for browser automation
* Railway for deployment
* No LLM used in the main workflow because most of the task was deterministic automation

1. Input:

   * user emails the agent
   * Resend receives the email and sends a webhook to my FastAPI app
   * the app gets the email subject/body from Resend

2. Extract:

   * matter number e.g. M12205
   * document type e.g. Other Documents

3. Validate:

   * matter number format is M + digits
   * document type is one of the expected categories
   * if not, it should go to error handling / clarification

4. Fetch:

   * open UARB page using Playwright
   * input the matter number into the Go Directly to Matter box
   * press Search
   * collect matter metadata:

     * title / description
     * type
     * category
     * date received
     * decision/final date
   * count number of documents from the tab labels:

     * Exhibits
     * Key Documents
     * Other Documents
     * Transcripts
     * Recordings
   * navigate to the requested document tab
   * press the Go GET IT button for each document
   * download up to 10 files
   * added scrolling because only some document rows show in the viewport at once
   * save files in a temporary folder while the request is being processed
   * compress downloaded documents into a ZIP

5. Email agent:

   * generate an email response using the metadata and counts from the page
   * attach the ZIP file
   * response includes:

     * matter number
     * metadata summary
     * count of all document categories
     * how many requested files were downloaded
     * mention that the downloaded files are attached as a ZIP

6. Deployment:

   * deployed the FastAPI app on Railway
   * used Resend webhook so the agent can receive emails without my laptop running
   * used Resend API to send the response email with the ZIP attachment
   * configured my domain/DNS so the agent email address can receive and send messages

7. Error handling / future improvements:

   * if matter number does not exist, send a clearer error response
   * if document type is invalid, send a clarification email listing valid options
   * add simple normalization/fuzzy matching for typos, such as “other docs” or “Other Docmets” → “Other Documents”
   * possibly use an LLM only for unclear natural language parsing, not for the core automation
